## 云开发环境 / DevContainers

- [gitpod-io/gitpod](https://github.com/gitpod-io/gitpod) - 云原生开发环境，开源核心。一键创建预配置开发环境。支持 SaaS + 自托管模式
- [kasmtech/KasmVNC](https://github.com/kasmtech/KasmVNC) - 容器化的桌面和工作区，支持完整浏览器桌面环境。可以提供完整桌面体验，类似 Manus ⭐⭐⭐⭐⭐
- [hashicorp/vagrant](https://www.vagrantup.com/) - QEMU + Vagrant，完整硬件虚拟化，经典虚拟机组合。可以创建完全一致的开发环境。适合需要完整操作系统的场景
- [lima-vm/lima](https://github.com/lima-vm/lima) - macOS 上运行 Linux 虚拟机，轻量级。开箱即用，与 Docker 兼容。适合 macOS 开发环境
- [canonical/multipass](https://github.com/canonical/multipass) - Canonical 出品，轻量级虚拟机管理。一键创建 Ubuntu 虚拟机，简单易用。适合快速测试环境
- [firecracker-microvm/firecracker](https://github.com/firecracker-microvm/firecracker) - AWS 开源的轻量级虚拟机Monitor。极快启动，适用于Serverless场景。适合微虚拟机、Serverless函数

## AI Agent / 精选项目

- [bytedance/deer-flow](https://github.com/bytedance/deer-flow) - DeerFlow 2.0，开源超级代理工具。将子代理、内存和沙箱组织在一起，配合可扩展技能，让代理可以完成几乎任何事情 ⭐
- [msitarzewski/agency-agents](https://github.com/msitarzewski/agency-agents) - 给 Claude Code / Cursor 准备的"AI 人才市场"——不用一个通用 AI 干所有活，让专业 AI 干专业活。开箱即用多个专业角色：前端专家、安全审计、创意激发、事实核查。纯提示词工程，直接寄生在 Claude Code / Cursor 使用。开源 2025-10-13，星标 ≈ 46k，本周增长 +23.2k ⭐ (增速第一)
- [obra/superpowers](https://github.com/obra/superpowers) - 一套 AI 编程 Agent 的"技能商店"和开发方法论，让 AI 写代码更靠谱。痛点：Claude Code / Copilot 经常上来就写代码，跳过关键步骤。superpowers 系统化流程：Understand 理解需求 → Plan 制定方案 → Implement 实现 → Verify 验证。提供 14 个核心技能，支持 Claude Code / Codex / Cursor。星标 ≈ 10.5k，本周增长 +19.2k ⭐ (增速第二)。社区评价：这不是一个工具，这是一套"AI 编程"的最佳实践沉淀
- [666ghj/MiroFish](https://github.com/666ghj/MiroFish) - 简洁通用的群体智能预测引擎——让多个 AI 一起预测，结果更准确。原理："三个臭皮匠，顶个诸葛亮"，多个 AI 模型协同预测比单个准。支持股票价格、体育比赛、选举结果、天气等各种预测场景。本周增长 +17.6k ⭐ (增速第三)
- [volcengine/OpenViking](https://github.com/volcengine/OpenViking) - 字节火山引擎开源的 AI Agent 上下文数据库——解决 Agent "记不住事"的痛点。AI Agent 最大痛点之一：记忆缺失，对话长了就忘记前文。OpenViking 把 Agent 的上下文变成结构化数据库，高效存储、快速检索。本周增长 +10.2k ⭐
- [lightpanda-io/browser](https://github.com/lightpanda-io/browser) - 专为 AI 设计的无头浏览器——给 AI 装上"眼睛"，让 AI 能真正上网。传统无头浏览器太笨重，启动慢资源消耗大。lightpanda 专为 AI 自动化浏览设计：启动快、资源占用小、适配 AI 抓取需求。让你的 AI Agent 真的能"上网看新闻""填表单""点击按钮"。本周增长 +9.9k ⭐
- [colorizeblob/impeccable](https://github.com/colorizeblob/impeccable) - AI 执行的安全检查层——给 AI 的操作系上"安全带"，防止乱删文件乱执行命令。AI Agent 能干活，但也容易"胡来"，一不小心删了你整个项目。impeccable 在 AI 和系统之间加一层安全校验：危险操作要确认，敏感文件不能碰，操作可回滚
- [coze-dev/page-agent](https://github.com/coze-dev/page-agent) - 网页 AI 自动化 Agent——让 AI 直接操作网页完成任务。AI 输入自然语言指令，Agent 自动在网页上点击、填写、提交。适合填表、爬信息、自动化办公流程
- [context-hub/context-hub](https://github.com/context-hub/context-hub) - AI 上下文仓库——帮你压缩、存储、复用上下文，省钱。现在大模型按 Token 收费，长上下文一分钟就是几毛钱，很多场景重复上下文每次都要重新发，浪费钱。context-hub 把公共上下文存在服务端，复用 Token，大幅降低成本
- [brexhq/deepagents](https://github.com/brexhq/deepagents) - 规模化多智能体开发框架——让你低成本开发、运行多个智能体。开发单个 Agent 简单，开发几十个 Agent 规模化就很复杂。deepagents 提供框架：标准化、模块化、方便编排，降低企业级多智能体落地成本
- [microsoft/BitNet](https://github.com/microsoft/BitNet) - 微软开源的 1-bit 大模型压缩技术——极致压缩，让大模型更便宜。传统大模型是 16bit/8bit，BitNet 1bit 就够了，极端压缩。模型体积缩小十几倍，推理速度超快，内存占用超低。让大模型能在更低成本下规模化部署，尤其是端侧设备
- [badlogic/pi-mono](https://github.com/badlogic/pi-mono) - 轻量级模块化 AI Agent 执行框架，OpenClaw 本身就是基于这个框架设计理念开发的。极简设计，模块化和解耦，适合本地运行 AI Agent，代码精简容易二次魔改，被 OpenClaw 借鉴了底层架构思想
- [justlovemaki/OpenClaw-Docker-CN-IM](https://github.com/justlovemaki/OpenClaw-Docker-CN-IM) - 打通了国产 IM（飞书/微信/钉钉）的 OpenClaw Docker 镜像，国内用户一键部署。现成 Docker 镜像，省去编译配置，已经打通国产即时通讯软件，国内网络优化拉取速度快
- [ValueCell-ai/ClawX](https://github.com/ValueCell-ai/ClawX) - OpenClaw 社区魔改版本。社区开发者针对特定场景做了功能增强，适合喜欢折腾的玩家体验新特性
- [OpenBMB/EdgeClaw](https://github.com/OpenBMB/EdgeClaw) - 清华开源 "断网也能干活的龙虾" —— 两栖版端侧 AI 智能体。两栖能力：联网能调用云端大模型，断网能本地运行。端侧优化资源占用小，完全离线也能干活，适合隐私敏感场景
- [shareAI-lab/learn-claude-code](https://github.com/shareAI-lab/learn-claude-code) - Claude Code 从入门到精通实战教程。真正理解 Agent Harness 工程，系统化学习怎么用好 Claude Code，沉淀了社区最佳实践，适合想提升 AI 编程效率的开发者
- [Agents365-ai/zotero-rag-cli](https://github.com/Agents365-ai/zotero-rag-cli) - Zotero 文献库 RAG 语义检索 CLI 工具。本地对你的 Zotero 文献库建立 RAG 索引，支持语义搜索和混合检索，用本地嵌入模型隐私安全，返回最相关的论文 + 匹配片段，配合 Claude Code 就能让 AI 引用你的文献写作文
- [Crosstalk-Solutions/project-nomad](https://github.com/Crosstalk-Solutions/project-nomad) - 自包含的离线生存计算机。独立的、离线优先的知识和 AI 系统，预装了关键工具、知识库和人工智能，让你随时随地获取信息和算力，不依赖公共互联网。适用：网络不可用环境、隐私敏感工作、极端情况离线生存，包含离线地图 + 离线维基百科 + 本地 AI
- [liliMozi/openhanako](https://github.com/liliMozi/openhanako)
- [docling-project/docling](https://github.com/docling-project/docling) - Docling 简化了文档处理，解析了各种格式（包括高级 PDF 理解），并与 gen AI 生态系统实现了无缝集成
